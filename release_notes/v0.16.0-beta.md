# 🪼 Kurage v0.16.0-beta リリースノート

プッシュ通知と DeckUI の不具合修正が中心のアップデートです。

- 配布: Android (Google Play クローズドテスト) / Windows ポータブル版 (zip) / Web https://kurage.demo2.jp
- これはベータ版です。仕様変更や、まれに再ログイン・データ消去が起こる場合があります。
- Windows 版は未署名のため、初回に「Windows によって PC が保護されました」と出たら「詳細情報」→「実行」で起動できます（同梱の「お読みください.txt」にも記載）。

## 🛠 改善・修正

- **Android のプッシュ通知が届かなくなる問題への対策を複数入れました**
  - v0.15.0-beta 更新後に通知が来なくなったという報告への対応です。
  - 通知アイコンの処理が原因で通知が表示されずに消えていた可能性があるため、失敗時は従来のアイコンで表示し直すようにしました。
  - 端末側の通知用トークンが切り替わったとき、これまではアプリを再起動するまで通知が止まっていましたが、自動で登録し直すようにしました。
  - 新しくログインしたアカウントは、その場でプッシュ通知の受信登録が行われるようになりました（従来は次回起動まで届きませんでした）。
- **設定に「プッシュ通知の状態を確認」を追加しました**（設定 → 通知）
  - 通知権限・接続状態・アカウントごとの登録結果をその場で確認でき、同時に受信登録もやり直します。**通知が届かないときは、まずこれをお試しください**。結果の画面を開発者に伝えてもらえると原因調査がスムーズです。
- **DeckUI で投稿画面を開閉するたびに通知カラムが再読込される問題を修正しました**
  - 投稿ペインの開け閉めで通知カラムがくるくる再読込され、タイムラインのスクロール位置もリセットされることがありました。開閉してもカラムの状態を保持するようになりました。

## 📝 既知の注意点

- プッシュ通知が引き続き届かない場合は、「設定 → 通知 → プッシュ通知の状態を確認」の表示内容を添えてご連絡ください。
