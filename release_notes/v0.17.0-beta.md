# 🪼 Kurage v0.17.0-beta リリースノート

通知アイコン問題の根本修正、タイムラインの「放置すると古い位置で止まる」問題の修正、そして投稿の公開範囲デフォルト設定への対応です。

- 配布: Android (Google Play クローズドテスト) / Windows ポータブル版 (zip) / Web https://kurage.demo2.jp
- これはベータ版です。仕様変更や、まれに再ログイン・データ消去が起こる場合があります。
- Windows 版は未署名のため、初回に「Windows によって PC が保護されました」と出たら「詳細情報」→「実行」で起動できます（同梱の「お読みください.txt」にも記載）。

## ✨ 新機能

- **投稿の公開範囲がアカウントのデフォルト設定に従うようになりました**
  - Mastodon 本体（Web 版など）で設定する「投稿の公開範囲のデフォルト」を、Kurage の新規投稿画面が初期値として反映します。これまでは常に「公開」固定でした。
  - 返信は従来どおり返信元の公開範囲を引き継ぎます。下書きの復元や投稿の編集も元の公開範囲を優先します。
  - 複数アカウントで同時投稿する場合は、各アカウントの設定のうち**最も狭い公開範囲**を初期値にします（意図せず広く公開してしまう事故を防ぐため）。
  - デフォルト公開範囲は Kurage からも変更できます:「設定 → アカウント設定」の各アカウントの 🌐 ボタンから。変更はサーバに保存されるので、Web 版など他のクライアントにも反映されます。

## 🛠 改善・修正

- **プッシュ通知のアイコンが表示できない問題を根本修正しました**
  - v0.16.1-beta の対策後も全端末でクラゲアイコンの読み込みに失敗していました。原因はリリースビルド時の「未使用リソース削除」の最適化が通知アイコンを誤って削除していたことで、削除されないよう保護指定を追加しました。今回のビルドにアイコンが含まれていることを確認済みです。
  - 「設定 → 通知 → プッシュ通知の状態を確認」のテスト通知で、クラゲのシルエットが表示されれば正常です。
- **タイムラインを最新位置で放置すると、いつのまにか古い投稿の位置で止まって未読がたまっていく問題を修正しました**
  - Android では省電力などの影響でストリーミング接続が裏で切れて再接続することがあり、そのたびに表示位置が「切断前の投稿」に固定されてしまうのが原因でした。最新位置（いちばん上）を見ている間は、再接続後も先頭に追従し続けます。
  - 途中までスクロールして読んでいる場合は、従来どおり読んでいる位置を保ったまま未読バッジが増えます。
