# 🪼 Kurage v0.18.0-beta リリースノート

Web 版に画像の保存ボタンを追加。また v0.17.0 の「タイムラインが古い位置で止まる」対策を見直し、アプリ復帰時に読んでいた位置へ正しく戻るようにしました。

- 配布: Android (Google Play クローズドテスト) / Windows ポータブル版 (zip) / Web https://kurage.demo2.jp
- これはベータ版です。仕様変更や、まれに再ログイン・データ消去が起こる場合があります。
- Windows 版は未署名のため、初回に「Windows によって PC が保護されました」と出たら「詳細情報」→「実行」で起動できます（同梱の「お読みください.txt」にも記載）。

## ✨ 新機能

- **Web 版: 画像ビューアに保存ボタンを追加しました**
  - 画像を開いて右上の保存ボタン（⬇）を押すと、ブラウザのダウンロードとして保存できます。
  - Web 版は描画方式の都合で右クリックの「名前を付けて画像を保存」が使えないため、この保存ボタンをご利用ください。
  - 一部のサーバー（画像 CDN の設定による）ではダウンロードの代わりに画像が新しいタブで開きます。その場合はタブ側で右クリック（スマホは長押し）から保存できます。

## 🛠 改善・修正

- **アプリ復帰時に、読んでいた位置から再開するようになりました**
  - v0.17.0 の「放置すると古い位置で止まる」対策に副作用があり、タイムラインの上の方を見た状態でアプリを離れて戻ると、読んでいた位置を飛ばして一気に最新へジャンプしてしまうことがありました。
  - 今回この方式を見直し、復帰時は**常に読んでいた位置を保ったまま**、その間の新着は「未読 N 件」バッジでお知らせするようにしました。バッジをタップすると最新までジャンプします。
- **「いつのまにか未読がたまる」の根本原因（接続切れの検知遅れ）を改善しました**
  - ストリーミング接続が裏で切れたことに気づくまで最大 10 分かかっていたのを、サーバーの生存信号（ハートビート）を監視して**約 1.5 分**で検知・再接続するようにしました。取り逃した投稿がまとめて未読に積まれる頻度が大きく減ります。
- **デスクトップ (Deck) 版: 返信をやめて新規投稿にしたとき、前の返信相手の @メンションが残る問題を修正しました**
  - あわせて、返信・引用・編集の作成中に「新規投稿」ボタンを押した場合は、いきなり破棄せず「続ける / 破棄して新規投稿」の確認を出すようにしました。
  - 投稿欄をピン固定している場合に、返信を投稿したあとも返信モードのまま残ってしまう（次の投稿が同じ相手への返信になってしまう）問題も修正しました。
