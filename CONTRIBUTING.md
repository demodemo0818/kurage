# Contributing to Kurage

Kurage への関心をありがとうございます。個人開発のプロジェクトなので体制は
小さめですが、バグ報告・提案・PR を歓迎します。

## バグ報告・機能提案

- GitHub Issue でお願いします。日本語・英語どちらでも構いません。
- バグ報告には「環境 (プラットフォーム / アプリのバージョン / 接続先サーバ)」
  「再現手順」「期待した挙動と実際の挙動」を含めてください。

## Pull Request

- **大きな変更 (新機能・大規模リファクタ) は、先に Issue で相談してください。**
  設計方針と合わない PR をお断りするのはお互いに不幸なので、着手前の一言が
  一番のショートカットです。誤字修正や明白なバグ修正のような小さな PR は
  いきなり送ってもらって大丈夫です。
- コードのアーキテクチャと既知の罠は [CLAUDE.md](CLAUDE.md) にまとまっています。
  特にタイムライン周り (SSE / スクロール) は罠が多いので、変更前に該当節を
  ご確認ください。
- 送る前に以下が通ることを確認してください:
  ```bash
  flutter analyze   # 0 issues を維持しています
  flutter test
  ```
