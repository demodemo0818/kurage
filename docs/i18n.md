# i18n (英語対応) の設計と移行ガイド

2026-07-19 に方針確定。目的は**海外ユーザー獲得**で、アプリ UI・README・
Play ストア文言・Web HTML・紹介サイトまで英語化する。**全文言の英訳完了まで
言語切替は解放しない** (段階移行・一括公開)。解放リリースは **v1.1.0**。

## 仕組み

- **gen_l10n** (Flutter 公式)。設定は [l10n.yaml](../l10n.yaml)、文言は
  [lib/l10n/app_ja.arb](../lib/l10n/app_ja.arb) (テンプレート) と
  [lib/l10n/app_en.arb](../lib/l10n/app_en.arb)。
- 生成物 `lib/l10n/gen/` は **git 管理外**。`flutter pub get` /
  `flutter gen-l10n` / ビルドで自動再生成される。
- アクセスは [lib/l10n/l10n.dart](../lib/l10n/l10n.dart) 経由:
  - Widget 内: `context.l10n.foo` (拡張メソッド)
  - **BuildContext が無い層** (services / 例外メッセージ / バックグラウンド
    通知): トップレベル変数 `l10n`。MyApp.build が locale 決定のたびに
    `updateGlobalL10n` で同期する。FCM バックグラウンド isolate では
    `updateGlobalL10nFromPrefs()` で保存済み設定から初期化すること
    (widget tree が無く MyApp が走らないため)。
- 表示言語設定は `settingsProvider.appLocale` ('system' / 'ja' / 'en'、
  既定 'system')。Locale への解決は `resolveAppLocale` (非対応言語は en に
  フォールバック)。**ただし全訳完了までは main.dart が locale を ja 固定に
  しており、この設定は UI からも変更不可** (ピッカー未実装)。

## 新規文言のルール

1. ユーザーに見える文言は日本語ハードコードせず、**ja/en 両方の .arb に
   同時追加**してから `context.l10n.key` / `l10n.key` で参照する。
   CI が未翻訳キー (ja にあって en に無い) を検出して fail する。
2. キー名は lowerCamelCase。画面固有なら接頭辞 (`compose…`, `nav…`) で揃える。
3. 数値を含む文言は placeholder + 英語側は ICU plural (`relativeMinutesAgo`
   参照)。語形が分岐する文言は ICU select (`composeInProgressTitle` 参照)。
4. **debugPrint 等の内部ログ・コードコメント・docs/ は日本語のまま**でよい。
5. 英訳の用語は **Mastodon 公式 Web UI の英語表記に準拠**する
   (boost / favourite / content warning / bookmark / mute / block など。
   "favorite" ではなく英式 "favourite")。
